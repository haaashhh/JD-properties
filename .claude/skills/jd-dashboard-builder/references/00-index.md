# Reference Index — Route Here First

| Topic | File | When to read |
|-------|------|-------------|
| Visual components, styling, layout, colors, charts, tables | `ui-patterns.md` | Any UI work |
| Supabase client, queries, mutations, real-time, uploads, auth | `supabase-patterns.md` | Any data operation |
| Forms, validation, server actions, optimistic updates, error handling | `form-patterns.md` | Any user input |
| Module-specific component recipes (deal form, kanban, budget table, etc.) | `module-recipes.md` | Implementing a named feature |

**Starting a new component?** Read `ui-patterns.md` first for styling conventions, then `module-recipes.md` for the specific recipe if one exists.

**Wiring up data?** Read `supabase-patterns.md` for the correct client pattern (server vs client), then `form-patterns.md` if the component involves user input.
